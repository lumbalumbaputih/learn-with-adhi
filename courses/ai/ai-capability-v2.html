The Landscape

30 min

Why people talk past each other

There isn't one "AI." There are tiers, and the gap between them is now enormous. Here's the mental model:

Group A — "AI is overhyped": Free tier, chatbots, voice mode. Judges AI by consumer chat and viral fails.

Group B — "AI is transformative": Paid frontier, agentic, code/research. Judges AI by professional output quality.

Both are right about what they've seen. The problem: they haven't seen the other side.

The three tiers you need to know

Tier 1: Consumer chat (free/low-cost). This is ChatGPT free, Gemini free, Copilot free. Good for casual Q&A, brainstorming, basic writing. These models are often older, rate-limited, and lack tool access. This is what most people have tried.

Tier 2: Paid frontier chat. ChatGPT Plus/Pro, Claude Pro. Better models (GPT-4o, Claude Sonnet/Opus), longer context, web search, file handling. This is where you currently are — it's a big step up, but it's still conversational.

Tier 3: Agentic frontier. Claude Code, OpenAI Codex, Devin, Cursor Agent mode. The model doesn't just answer — it gets a terminal, a file system, tools. It plans, executes, iterates, debugs. It works autonomously for minutes to hours. This is where the dramatic jump is.

Where you are

You're at Tier 2, dipping a toe into Tier 3 via your Toko Curah project (GitHub/Vercel). This course is about making that jump deliberate.

Why coding/math improved faster than writing

To train a model to be better, you need a way to score its outputs. For code: did it pass the tests? Yes or no. For math: is the answer correct? Yes or no. This clear signal lets you run reinforcement learning — the model tries thousands of approaches and learns from what worked.

For writing, advice, or creative tasks? There's no unit test. "Good writing" is subjective. So training is slower, messier, and relies on human ratings (expensive, inconsistent). This is why the coding/reasoning capabilities have sprinted ahead while chat quality has improved more gradually.

Key Concepts

25 min

Reinforcement Learning (RL) — the engine behind the jump

Standard AI training: show the model billions of examples of text, it learns patterns. This is pre-training. It makes the model knowledgeable but not strategic.

RL adds a second phase: the model tries things, gets scored, and adjusts. Like a student who doesn't just read textbooks but does practice exams with answer keys.

The breakthrough: when OpenAI and Anthropic started applying RL to coding and reasoning tasks with automated scoring (unit tests, proof checkers), performance jumped dramatically. Models went from "sometimes helpful" to "reliably solves hard problems."

Agentic AI — what changes when the model gets tools

A chatbot answers questions. An agent takes actions. The difference:

Chat: You ask "how do I fix this bug?" → model suggests code → you copy, paste, test, come back with errors → repeat.

Agent: You say "fix this bug" → model reads the codebase, writes a fix, runs the tests, sees they fail, tries a different approach, runs tests again, they pass, commits the change. All without you.

Real example

Claude Code can be given a GitHub issue like "add dark mode to the settings page" and will: read the relevant files, understand the CSS architecture, implement the toggle, update the state management, test it, and submit a pull request. Tasks that took a junior developer a day now take minutes.

Context windows & tool use — why size matters

Context window = how much text the model can "see" at once. Early GPT-3: ~4,000 tokens (~3 pages). Current frontier: 100K–200K tokens (a full novel). This means the model can read your entire codebase or entire research paper collection before answering.

Tool use = the model can call external functions: run code, search the web, read files, call APIs. This is what turns a chatbot into an agent.

The "vibe coding" phenomenon

A new term for 2025: non-programmers describing what they want in plain English and having AI write working software. It's real, but with caveats — the AI can build the thing, but you still need to understand what you're asking for, evaluate what it built, and debug when things go sideways.

The Models

15 min

The main players (as of early 2026)

Anthropic (Claude) — Claude Opus 4.6, Sonnet 4.6, Haiku 4.5. You're using Opus right now. Anthropic's strength: long-context reasoning, careful instruction-following, Claude Code (agentic coding). Strong safety focus.

OpenAI (GPT / Codex) — GPT-4o for consumer, o-series (o1, o3, o4-mini) for reasoning, Codex for agentic coding. Codex is their answer to Claude Code — it runs in a sandbox and tackles full repos. Largest user base by far.

Google DeepMind (Gemini) — Gemini 2.5 Pro/Flash. Huge context windows (up to 1M tokens). Strong multimodal (text + image + video + code). Integrated with Google ecosystem.

Meta (Llama) — Open-weight models. Llama 4 series. Free to download and run locally. Important for the ecosystem but not the frontier of agentic capability.

Others: Mistral (EU-based), xAI (Grok), various Chinese labs (DeepSeek, Qwen). DeepSeek's R1 model was a notable 2025 moment — competitive reasoning at low cost.

The tools that matter for you

Already using well: Claude Pro (Tier 2 chat) for writing, research, document work. Keep doing this.

Next step: Claude Code — a command-line tool where Claude operates your computer to build/edit projects. This is what you need for Toko Curah and future web projects.

Be aware of: Cursor (AI-native code editor), GitHub Copilot (inline code suggestions), Windsurf. These are useful for daily coding but Claude Code is the most powerful agentic option right now for your use case.

Hands-On Project

90 min

Exercise 1: The same task at three tiers

Pick a task you've done recently — say, "build a simple landing page with a contact form." Now do it three ways:

  1. Tier 1: Go to chatgpt.com without logging in (free tier). Ask it to build you a landing page. Copy-paste what it gives you. Note what works, what doesn't, how much manual fixing you need.
  2. Tier 2: Ask me (Claude Pro) the same thing in this chat. Compare quality, whether I ask clarifying questions, how complete the output is.
  3. Tier 3 (demo): Watch a Claude Code demo video of someone building something similar. Notice: the model reads files, writes code, runs it, sees errors, fixes them — no human in the loop.

What you'll notice

Tier 1 → 2 is a quality jump. Tier 2 → 3 is a category jump. It's not "better answers" — it's autonomous work.

Exercise 2: Prompt engineering — make Tier 2 punch above its weight

Before you jump to Tier 3 tools, learn to extract maximum value from Tier 2 (what you're already paying for). Try these prompt patterns:

  1. Chain of thought: Add "Think step by step" or "Break this down before answering." Compare the output quality.
  2. Role + constraints: "You are a senior environmental researcher. Write for an IELTS 7 academic audience. Use Chicago 17th author-date citations."
  3. Iterative refinement: Instead of one big prompt, have a conversation. First ask for an outline, critique it, then expand sections. You'll get much better results than a single monolithic request.

Exercise 3: Spot the hallucination

Ask any AI model (free or paid) to cite 5 academic papers on circular economy in construction with DOIs. Then verify each DOI at doi.org. Count how many are real. This is the #1 skill for your academic work — knowing when to verify.

Reality check

Even frontier models hallucinate citations. The difference: frontier models do it less and can use tools (web search) to self-verify. But you should always verify academic citations regardless of which model generated them.

Claude Code Setup

150 min

Prerequisites

Before starting, you need:

  1. Node.js installed on your machine (you may already have this from the Toko Curah project)
  2. A terminal you're comfortable with (Terminal on Mac, or WSL/PowerShell on Windows)
  3. A Claude API key or Claude Pro/Max subscription — Claude Code connects to the API

Honest assessment

You're still building comfort with Git and terminal basics. That's fine — Claude Code actually helps with this because it explains what it's doing as it works. But expect a learning curve in the first session. Don't get discouraged.

Step 1: Install Claude Code

Open your terminal and run:

npm install -g @anthropic-ai/claude-code

Then navigate to a project folder (e.g., your Toko Curah repo) and run:

claude

This launches Claude Code in your terminal. It can now see your files, run commands, and edit code — with your approval at each step.

Step 2: Your first agentic task

Try something real with Toko Curah. In the Claude Code terminal, type something like:

"Look at the current landing page. Add a product section that shows 3 product cards with images, names and prices. Make it responsive and match the existing design."

Then watch. Claude Code will:

  1. Read your existing files to understand the project structure
  2. Identify the tech stack (HTML? React? Next.js?)
  3. Write the code
  4. Ask for your approval before making changes

Key insight

You don't need to know how to write the code. You need to know what you want and be able to evaluate whether what it built is correct. That's "vibe coding" — and your domain knowledge of the business is the irreplaceable part.

What to watch for

Approval prompts: Claude Code asks before running commands or editing files. Read what it's proposing. This is how you learn what's happening.

Over-engineering: Sometimes it'll build something more complex than needed. Don't be afraid to say "simpler, just do X."

Context: The more of your project it can see, the better. Working in the project root directory helps.

Staying Current

ongoing

The core problem: AI moves faster than any individual can track

New models drop monthly. New tools weekly. You can't read everything. Here's how to stay calibrated without it becoming a full-time job.

Build a minimal information diet

Weekly (15 min): Skim one of these for headlines:

  • One Useful Thing — Ethan Mollick's blog. Practical, grounded, academic perspective.
  • Simon Willison's blog — Technical but readable. Best for tool/product updates.

Monthly (1 hour): Try one new AI tool or feature you haven't used before. Could be a new Claude feature, a Cursor experiment, or testing a competitor.

Quarterly: Revisit this course. Ask me "what's changed in AI since [date]?" and I'll update you.

The mindset that matters

Don't try to become an AI researcher. Your edge is domain expertise + AI fluency. A marine engineer / environmental researcher who can wield agentic AI tools effectively is far more valuable than someone who only knows AI but has no domain.

Your specific advantage

You understand circular economy, building deconstruction, sustainability policy, aquaculture, and e-commerce operations. None of the AI engineers building these tools know these domains. Your job is to be the person who can direct the tools toward real problems in your fields. That's the skill set that will be in highest demand.

What to be skeptical about

AGI timelines. People claiming AGI in 2–3 years are speculating. Progress is real but unpredictable. Plan for steady capability gains, not a sudden singularity.

"AI will replace X." It's replacing tasks, not jobs (for now). The people getting replaced are those who refuse to learn the tools, not those in specific professions.

Benchmarks. When a lab says "our model scores 95% on [benchmark]" — benchmarks are gamed, saturated, and often don't reflect real-world usefulness. Judge by trying the tool yourself.
