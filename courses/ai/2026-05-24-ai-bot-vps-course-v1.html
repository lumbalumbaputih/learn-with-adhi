MODULE 01 / 08

The Restaurant Analogy

Before writing a single line of code, you need to understand what you're building. We'll use one analogy throughout this entire course — and it will make everything click.

Learning Objectives

  • Understand what a VPS, OpenRouter, and Telegram bot each do
  • See how they connect together using the restaurant analogy
  • Know the four pieces you're about to build

Your Restaurant, Your Rules

Imagine you want to open a restaurant. You don't own a building yet, you don't have chefs, and you have no dining room. That's exactly where you are right now. This course gives you all three.

The four pieces of your AI bot system

The Four Pieces

VPS (Virtual Private Server) — This is your kitchen. A small computer rented in a data centre that runs 24/7 even when your laptop is off. Your bot code lives here.

OpenRouter — This is the menu of chefs you can hire. Instead of picking one AI company and sticking with it, OpenRouter lets you send requests to GPT-4o, Claude, Gemini, and hundreds more — all through one single address. You pay per meal (per token).

Your Python Bot — This is the restaurant manager. It receives orders from customers (Telegram messages), sends them to the chosen chef (OpenRouter), and delivers the food back (AI reply).

Telegram — This is your dining room. It's where your customers (users) sit and chat with your bot. Free, works everywhere, and has a dead-simple bot API.

Why Telegram and not WhatsApp?
WhatsApp's business API costs money and requires Meta approval. Telegram is free, instant, and bots work on the same account normal users have. For learning, it's the obvious choice.
Analogy Cheat Sheet
VPS = Kitchen building  ·  OpenRouter = Chef hire menu  ·  Python code = Restaurant manager  ·  Telegram = Dining room  ·  API key = Chef's employment contract  ·  Token = One word of food

Rent a Kitchen

A VPS is just a small computer you rent by the month. It lives in a data centre and never turns off. This module gets you one set up and ready.

Learning Objectives

  • Create a DigitalOcean account and deploy a Droplet
  • Pick the right size for a bot (spoiler: the smallest one)
  • Understand what you're paying for
$6
per month — cheapest DigitalOcean Droplet (1 vCPU, 1GB RAM, 25GB SSD)
24/7
uptime — your bot answers even at 3am while you sleep
~2 min
to spin up a new Droplet from scratch on DigitalOcean

Step-by-Step: Create Your Droplet

  • Go to cloud.digitalocean.com and create a free account. You'll get a $200 credit for 60 days — enough to run this entire project for free while learning.

  • Click Create → Droplets in the top menu.

  • Choose Region: pick the one closest to your users. For Indonesia-facing bots, Singapore (SGP1) is ideal.

  • Choose Image → Ubuntu 22.04 LTS. This is the most stable, widely documented Linux version for beginners.

  • Choose Size → Basic → Regular → $6/mo (1 vCPU, 1 GB RAM). More than enough for a Telegram bot.

  • Under Authentication, choose Password for now (simpler). Set a strong password — you'll use it to log in.

  • Click Create Droplet. In about 60 seconds, you'll see a green IP address. That's your kitchen's address.

What's an IP address?
Think of it as your kitchen's street address. When your bot needs to talk to Telegram or OpenRouter, they need to know where to deliver the response. That IP number is how the internet finds your VPS.

Choosing the Right Size

Start small. You can resize up later with one click.
Don't skip this
After creating your Droplet, go to Backups in the left sidebar and enable weekly backups (+$1.20/mo). If you break something, you can restore to last week instantly. Worth it.

First Day In

SSH is the key to your kitchen. Once inside, you'll learn the five Linux commands you'll use 90% of the time.

Learning Objectives

  • Log into your VPS using SSH
  • Update the server and install Python
  • Navigate around with basic Linux commands

Logging In: SSH

SSH (Secure Shell) is how you type commands on your VPS from your laptop. Think of it as a walkie-talkie that lets you give instructions to your kitchen remotely.

Open your laptop's terminal (Mac: search "Terminal" · Windows: install Windows Terminal from the Store) and type:

your laptop terminal
ssh root@YOUR_DROPLET_IP

Replace YOUR_DROPLET_IP with the number DigitalOcean showed you. It'll ask for your password — type it (you won't see it appear, that's normal) and hit Enter.

You're in!
When you see root@ubuntu-s-1vcpu-1gb:~# you've entered your kitchen. Every command you type now runs on the VPS, not your laptop.

The Five Commands You Need

Run the update sequence immediately after first login — takes ~2 minutes

Create Your Bot Folder

inside your VPS
mkdir /root/mybot
cd /root/mybot
ls   # should show empty — that's fine

This is your bot's home. Every file we create will go here.

Never close terminal mid-task
If your SSH connection drops while a command is running, it stops. In Module 07 we'll fix this permanently with systemd. For now, if something goes wrong, just SSH back in.

MODULE 04 / 08

Hire the Chef

OpenRouter is where you get access to AI models. Think of it as a staffing agency — you call them, tell them what kind of chef you need, they connect you, and you pay per dish cooked.

Learning Objectives

  • Create an OpenRouter account and get an API key
  • Understand what models are available and what they cost
  • Know which model to pick for a beginner bot

Get Your API Key

  • Go to openrouter.ai and sign up (free).

  • Click your profile → Keys → Create Key. Name it "telegram-bot".

  • Copy the key immediately — it starts with sk-or-v1-... and you can't see it again after closing the window.

  • Add credit: go to Credits → add $5. That's roughly 33 million tokens on GPT-4o-mini — enough for months of casual use.

Treat your API key like a password
Never paste it into a public GitHub repo or share it in a chat. Anyone who has it can spend your credits. We'll store it safely as an environment variable — not hardcoded in your script.

Choosing a Model

Token = roughly one word. A typical chat message is ~50–200 tokens.
What is a token?
One token ≈ one word (roughly). When your user says "What is the weather today?" that's 6 tokens in. When the bot replies with a 100-word answer, that's ~100 tokens out. Out-tokens cost more. Keep your system prompt short and your bot concise to save money.

Set Your API Key on the VPS

inside your VPS (paste your real key)
export OPENROUTER_API_KEY="sk-or-v1-your-key-here"
# To make it permanent across reboots:
echo 'export OPENROUTER_API_KEY="sk-or-v1-your-key-here"' >> ~/.bashrc
source ~/.bashrc

MODULE 05 / 08

Write the Manager

This is the Python script that sits in your VPS kitchen, takes orders from Telegram, calls OpenRouter, and delivers the AI's reply back. We'll build it line by line.

Learning Objectives

  • Install the required Python packages on your VPS
  • Understand each section of the bot script
  • Create the full bot.py file

Install Packages

on your VPS
cd /root/mybot
pip3 install openai python-telegram-bot --break-system-packages
Why openai package for OpenRouter?
OpenRouter is intentionally designed to be a drop-in replacement for the OpenAI API. You use the same openai Python library — you just change the base_url. This means switching models is one line of code.

The Full Bot Script

Create the file: nano /root/mybot/bot.py — then paste everything below.

bot.py — full script
import os
from openai import AsyncOpenAI
from telegram import Update
from telegram.ext import (
    Application, CommandHandler,
    MessageHandler, filters, ContextTypes
)

# ── 1. CONFIG ────────────────────────────────────────────
OPENROUTER_API_KEY = os.environ["OPENROUTER_API_KEY"]
TELEGRAM_TOKEN     = os.environ["TELEGRAM_TOKEN"]
MODEL = "openai/gpt-4o-mini"  # swap to any OpenRouter model

# ── 2. PERSONALITY ───────────────────────────────────────
# Change this to make your bot behave differently
SYSTEM_PROMPT = """You are a friendly, helpful AI assistant.
Keep answers concise (2-3 sentences unless asked for more).
Be warm and conversational, not robotic."""

# ── 3. MEMORY ────────────────────────────────────────────
# Each user gets their own conversation history
conversations = {}

# ── 4. AI CLIENT ─────────────────────────────────────────
client = AsyncOpenAI(
    base_url="https://openrouter.ai/api/v1",
    api_key=OPENROUTER_API_KEY,
)

# ── 5. HANDLERS ──────────────────────────────────────────
async def start(update: Update, ctx: ContextTypes.DEFAULT_TYPE):
    """Runs when user types /start"""
    await update.message.reply_text(
        "👋 Hi! I'm your AI assistant. Ask me anything!"
    )

async def handle_message(update: Update, ctx: ContextTypes.DEFAULT_TYPE):
    """Runs on every normal text message"""
    user_id  = update.effective_user.id
    user_msg = update.message.text

    # Build or retrieve this user's history
    if user_id not in conversations:
        conversations[user_id] = [
            {"role": "system", "content": SYSTEM_PROMPT}
        ]

    # Add what the user just said
    conversations[user_id].append(
        {"role": "user", "content": user_msg}
    )

    # Trim to last 20 exchanges (saves tokens)
    if len(conversations[user_id]) > 21:
        conversations[user_id] = (
            [conversations[user_id][0]]   # keep system prompt
            + conversations[user_id][-20:]
        )

    # Ask the AI
    response = await client.chat.completions.create(
        model=MODEL,
        messages=conversations[user_id],
    )
    reply = response.choices[0].message.content

    # Remember what the bot said
    conversations[user_id].append(
        {"role": "assistant", "content": reply}
    )

    await update.message.reply_text(reply)

# ── 6. MAIN ──────────────────────────────────────────────
def main():
    app = Application.builder().token(TELEGRAM_TOKEN).build()
    app.add_handler(CommandHandler("start", start))
    app.add_handler(
        MessageHandler(filters.TEXT & ~filters.COMMAND, handle_message)
    )
    print("Bot is running...")
    app.run_polling()

if __name__ == "__main__":
    main()

What Each Section Does

Memory (section 3) is what makes this a true agent — it remembers context across messages

MODULE 07 / 08

Keep the Lights On

Right now your bot dies the moment you close your laptop. We'll fix this with systemd — Linux's built-in process manager — so your bot runs forever and restarts automatically if it crashes.

Learning Objectives

  • Create a systemd service file for your bot
  • Enable the service to auto-start on reboot
  • Check logs when something goes wrong

The Service File

A systemd service file tells Linux: "here's a program, keep it running always, restart it if it crashes, and start it when the VPS boots."

create this file: nano /etc/systemd/system/mybot.service
[Unit]
Description=My Telegram AI Bot
After=network.target

[Service]
Type=simple
User=root
WorkingDirectory=/root/mybot
EnvironmentFile=/root/mybot/.env
ExecStart=/usr/bin/python3 /root/mybot/bot.py
Restart=always
RestartSec=10

[Install]
WantedBy=multi-user.target

Notice we switched to an EnvironmentFile. Create that file now:

create: nano /root/mybot/.env
OPENROUTER_API_KEY=sk-or-v1-your-key-here
TELEGRAM_TOKEN=1234567890:your-token-here
Secure the .env file
Run this so only root can read it: chmod 600 /root/mybot/.env

Enable and Start

on your VPS
# Tell systemd about the new service
systemctl daemon-reload

# Enable auto-start on reboot
systemctl enable mybot

# Start it now
systemctl start mybot

# Check it's running
systemctl status mybot

You should see Active: active (running) in green. Your bot now runs forever.

Useful Commands

Whenever you edit bot.py, run systemctl restart mybot to apply changes
You're done! Your bot is now production-ready
It starts on boot, restarts on crash, runs with no terminal open, and logs everything. This is exactly how real production services are run on Linux.

Cost-Saving Tips

Keep it cheap without losing quality

• Short system prompt — every message sends the full system prompt. 50 words > 500 words.

• Trim history — our script already limits to 20 messages. Consider 10 for cheaper bots.

• Gemini Flash — if GPT-4o-mini feels expensive, swap to google/gemini-flash-1.5 (half the price, still excellent).

• Set a monthly limit — in OpenRouter dashboard → Settings, set a hard spend cap so you never get surprised.

Reality check for a personal project
10 users × 5 messages × 150 tokens = 7,500 tokens/day. At $0.60/1M out tokens, that's about $0.0045/day in AI costs — less than half a cent. For most personal or small community bots, your VPS ($6/mo) will be your biggest expense by far.
